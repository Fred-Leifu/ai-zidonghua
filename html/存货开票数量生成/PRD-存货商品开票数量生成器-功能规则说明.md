# 存货商品开票数量生成器 - 功能规则说明

---

## 1. 需求概述

### 1.1 业务背景
当前系统生成的开票数量过于平均，缺乏随机性，无法满足实际业务需求。需要开发一个工具，根据存货数量和订单数量，自动生成无规律、符合业务逻辑的开票数量分配方案。

### 1.2 核心目标
- 生成的开票数量需围绕平均值随机分布
- 避免出现连续三个及以上相同的数量
- 确保每个订单至少分配1条
- 总数严格等于存货数量

---

## 2. 功能规则

### 2.1 输入参数

| 参数 | 类型 | 说明 | 约束条件 |
|------|------|------|----------|
| 存货数量(S) | 正整数 | 待分配的总数量 | S ≥ N |
| 订单数量(N) | 正整数 | 需要分配的订单数 | N ≥ 1 |

### 2.2 核心算法逻辑

#### 步骤1：初始化分配
- **规则**：每个订单先分配1条
- **目的**：确保每个订单至少有值
- **示例**：100条存货，20个订单 → 每单先得1条，剩余80条

#### 步骤2：正态分布随机分配（带相邻校验）
- **规则**：围绕平均值（S/N）进行正态分布随机生成
- **关键校验**：生成时检查与前两个订单数量是否相同，若相同则重新生成（最多尝试50次）
- **目的**：数量集中在平均值附近，同时避免连续三个相同值
- **示例**：平均值50，大部分订单数量在40-60之间，且不会出现"50,50,50"

#### 步骤3：总和精确调整
- **规则**：计算当前总和与目标存货数量的差值，逐个调整订单数量
- **约束**：调整时仍需检查是否会产生连续三个相同值
- **目的**：确保最终总和严格等于存货数量

#### 步骤4：结果输出
- **规则**：生成可复制的结果列表
- **格式**：每行一个数字，可直接粘贴到Excel

### 2.3 约束条件

| 约束 | 说明 | 验证方式 |
|------|------|----------|
| 最小值约束 | 每单至少1条 | 算法初始化保证 |
| 总和约束 | 总数 = 存货数量 | 最终校验确保 |
| 连续性约束 | 无连续三个相同值 | 生成时实时校验 |
| 随机性约束 | 围绕平均值正态分布 | 算法保证 |

---

## 3. 业务规则

### 3.1 校验规则
```
1. 存货数量和订单数量必须 > 0 且为整数
2. 存货数量必须 ≥ 订单数量（每单至少1条）
3. 不符合规则时提示对应错误信息
```

### 3.2 生成规则
```
1. 平均值 = 存货数量 ÷ 订单数量（四舍五入）
2. 数量围绕平均值正态分布
3. 禁止连续三个订单数量相同
4. 最终总和必须严格等于存货数量
```

---

## 4. 输出结果

### 4.1 输出格式

| 输出项 | 说明 |
|--------|------|
| 订单列表 | 每个订单的开票数量 |
| 订单总数 | 参与分配的订单数 |
| 合计数量 | 所有订单数量之和（=存货数量） |
| 平均数量 | 订单数量的平均值 |

### 4.2 输出示例

**输入**：存货数量=1000，订单数量=20  
**输出**：
```
订单总数：20
合计数量：1000
平均数量：50.0

订单明细：
1: 48条
2: 52条
3: 49条
4: 51条
5: 50条
...（后续订单随机分布在40-60之间）
```

---

## 5. 验收标准

### 5.1 功能验收

| 验收项 | 验收标准 |
|--------|----------|
| 输入校验 | 非正整数输入应提示错误 |
| 最小值约束 | 所有订单数量 ≥ 1 |
| 总和约束 | 所有订单数量之和 = 存货数量 |
| 连续性约束 | 不存在连续三个相同数量 |
| 随机性 | 多次生成结果应不同 |

### 5.2 性能验收
- 单次生成时间 ≤ 1秒
- 支持订单数量最大1000
