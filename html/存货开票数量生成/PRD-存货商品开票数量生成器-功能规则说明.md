# 存货商品开票数量生成器 - 功能规则说明

---

## 1. 需求概述

### 1.1 业务背景
当前系统生成的开票数量过于平均，缺乏随机性，无法满足实际业务需求。需要开发一个工具，根据存货数量和订单数量，自动生成无规律、符合业务逻辑的开票数量分配方案。

### 1.2 核心目标
- 生成的开票数量需围绕平均值随机分布
- 避免出现连续三个及以上相同的数量
- 确保每个订单至少分配1条
- 总数严格等于存货数量

---

## 2. 功能规则

### 2.1 输入参数

| 参数 | 类型 | 说明 | 约束条件 |
|------|------|------|----------|
| 存货数量(S) | 正整数 | 待分配的总数量 | S ≥ N |
| 订单数量(N) | 正整数 | 需要分配的订单数 | N ≥ 1 |

### 2.2 核心算法逻辑

#### 步骤1：正态分布随机分配（带相邻校验）
- **规则**：使用 Box-Muller 变换实现正态分布随机生成，围绕平均值（S/N）进行分配
- **标准差**：平均值的 1/3
- **关键校验**：生成时检查与前两个订单数量是否相同，若相同则重新生成（最多尝试100次）
- **最小值保证**：每个订单至少分配1条
- **目的**：数量集中在平均值附近，同时避免连续三个相同值

#### 步骤2：总和精确调整
- **规则**：计算当前总和与目标存货数量的差值，逐个调整订单数量
- **调整策略**：随机打乱候选顺序，优先选择不会产生连续三个相同值的位置进行调整
- **强制调整**：若无法找到安全调整位置，则强制调整以保证总和正确（总和优先）
- **目的**：确保最终总和严格等于存货数量

#### 步骤3：结果输出
- **规则**：生成可复制的结果列表
- **格式**：每行一个数字，可直接粘贴到Excel

### 2.3 约束条件

| 约束 | 说明 | 验证方式 |
|------|------|----------|
| 最小值约束 | 每单至少1条 | `Math.max(1, value)` 强制保证 |
| 总和约束 | 总数 = 存货数量 | 最终校验确保 |
| 连续性约束 | 无连续三个相同值 | 生成时实时校验（最多100次尝试） |
| 随机性约束 | 围绕平均值正态分布 | Box-Muller变换实现 |

---

## 3. 业务规则

### 3.1 校验规则
```
1. 存货数量和订单数量必须 > 0 且为整数
2. 存货数量必须 ≥ 订单数量（每单至少1条）
3. 不符合规则时提示对应错误信息："存货商品数量/订单数量不能为0、负数、小数"
```

### 3.2 生成规则
```
1. 平均值 = 存货数量 ÷ 订单数量（四舍五入）
2. 数量围绕平均值正态分布，标准差为平均值的1/3
3. 禁止连续三个订单数量相同（生成时校验）
4. 最终总和必须严格等于存货数量
5. 调整差值时优先选择不产生三连的位置，若无则强制调整
```

---

## 4. 输出结果

### 4.1 输出格式

| 输出项 | 说明 |
|--------|------|
| 订单列表 | 每个订单的开票数量 |
| 订单总数 | 参与分配的订单数 |
| 合计数量 | 所有订单数量之和（=存货数量） |
| 平均数量 | 订单数量的平均值（保留1位小数） |

### 4.2 输出示例

**输入**：存货数量=1000，订单数量=20  
**输出**：
```
订单总数：20
合计数量：1000
平均数量：50.0

订单明细：
1: 48条
2: 52条
3: 49条
4: 51条
5: 50条
...（后续订单随机分布在40-60之间）
```

### 4.3 复制功能
- 支持一键复制所有订单数量到剪贴板
- 格式为换行分隔，可直接粘贴到Excel

## 5. 技术实现

### 5.1 核心算法

| 算法 | 用途 | 实现方式 |
|------|------|----------|
| Box-Muller变换 | 生成正态分布随机数 | `Math.sqrt(-2.0 * Math.log(u1)) * Math.cos(2.0 * Math.PI * u2)` |
| Fisher-Yates洗牌 | 随机打乱候选数组 | 从后向前交换位置 |
| 三连检查 | 检测连续三个相同值 | 检查左、右及左右组合三种情况 |

### 5.2 关键函数

| 函数名 | 功能说明 | 参数 | 返回值 |
|--------|----------|------|--------|
| `generateQuantities(S, N)` | 核心生成算法 | S:存货数量, N:订单数量 | `{quantities[], sum, hasWarning}` |
| `normalRandom(mean, stdDev)` | 正态分布随机数生成 | mean:均值, stdDev:标准差 | 随机数 |
| `checkTriple(arr, idx, newValue)` | 检查三连 | arr:数组, idx:位置, newValue:新值 | boolean |
| `shuffleArray(array)` | 数组洗牌 | array:待打乱数组 | 无（原地修改） |
| `validate(stockQty, orderQty)` | 参数验证 | stockQty, orderQty | `{valid, message}` |

## 6. 验收标准

### 6.1 功能验收

| 验收项 | 验收标准 |
|--------|----------|
| 输入校验 | 非正整数输入应提示错误信息 |
| 最小值约束 | 所有订单数量 ≥ 1 |
| 总和约束 | 所有订单数量之和 = 存货数量 |
| 连续性约束 | 不存在连续三个相同数量（生成时最多尝试100次） |
| 随机性 | 多次生成结果应不同，围绕平均值分布 |
| 复制功能 | 点击复制按钮可将结果复制到剪贴板 |

### 6.2 性能验收
- 单次生成时间 ≤ 1秒
- 支持订单数量最大1000
