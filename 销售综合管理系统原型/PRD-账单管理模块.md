## 1. 产品概述

### 1.1 项目背景

随着公司业务规模的扩大，现有的账单管理方式已无法满足高效运营的需求。需要建立一套完整的账单管理系统，实现从出库单到账单的自动化流转，提高账务处理效率，减少人工操作错误。

### 1.2 产品目标

- 实现账单的自动化生成和管理
- 支持灵活的账期配置
- 提供完整的账单状态流转机制
- 支持出库单的灵活调整和异常处理
- 提供批量操作能力，提高工作效率
- 支持客户、课题组、单位三个维度的账单查看与分析

### 1.3 用户角色

| 角色名称 | 角色描述 |
|----------|----------|
| 商务中心-账款组 | 负责账单管理、异议处理、客户提醒等核心操作 |
| 其他业务人员 | 查看账单信息、导出数据等 |

## 3. 页面结构

### 3.1 页面层级关系（已实现）

```
账单管理模块
├── 左侧边栏菜单
│   ├── 👤 客户账单
│   ├── 👥 课题组账单
│   ├── 🏢 单位账单
│   ├── 📋 全部账单
│   └── ⚠️ 异议账单
│
├── 维度详情页（客户/课题组/单位）
│   ├── 统计卡片区
│   ├── 账期账单列表
│   └── 返回按钮 → 返回列表页
│
└── 账单详情页
    ├── 账单基本信息
    ├── 出库单列表
    ├── 操作历史
    └── 返回按钮 → 返回维度详情页
```

### 3.2 页面导航规则（已实现）

1. **列表页 → 详情页**：点击"查看账单"按钮进入对应维度详情页
2. **详情页 → 账单详情**：点击账期卡片进入账单详情页
3. **账单详情 → 详情页**：点击返回按钮返回对应维度详情页
4. **详情页 → 列表页**：点击返回按钮返回对应维度列表页

## 4. 业务流程

### 4.1 账单生成流程

```
出库单交付签收 → 第二天0点自动生成暂估账单 → 账单确认 → 后续状态流转
```

### 4.2 账单状态流转图

```
暂估账单
    ↓
待确认账单 ←────────────┐
    ↓                     │
已确认账单                │ 异议处理
    ↓                     │
待开票                   │
    ↓                     │
待回款                   │
    ↓                     │
已回款                   │
                          │
异议账单 ────────────────┘
```

### 4.3 出库单调整流程

```
选择出库单 → 执行操作（计入下期/标记不计费） → 填写原因 → 上传凭证（可选） → 确认提交 → 操作成功
```

### 4.4 多维度查看流程

```
客户账单列表 → 查看账单 → 客户详情页 → 账期列表 → 账单详情页
课题组账单列表 → 查看账单 → 课题组详情页 → 账期列表 → 账单详情页
单位账单列表 → 查看账单 → 单位详情页 → 账期列表 → 账单详情页
```

## 5. 数据模型

### 5.1 账单数据结构（已实现）

```javascript
{
  id: String,           // 账单编号
  customerId: String,   // 客户ID
  groupId: String,     // 课题组ID
  companyId: String,    // 单位ID
  customer: String,     // 客户名称
  period: String,       // 账期
  amount: Number,        // 账单金额
  status: String,       // 账单状态
  statusText: String,  // 状态文本
  createTime: String,   // 创建时间
  editable: Boolean,    // 是否可编辑
  disputeReason: String // 异议原因（仅异议账单）
}
```

### 5.2 出库单数据结构（已实现）

```javascript
{
  id: String,           // 出库单号
  billId: String,       // 关联账单ID
  date: String,         // 出库日期
  amount: Number,       // 金额
  status: String,       // 状态（normal/exclude/next-period）
  statusText: String    // 状态文本
}
```

### 5.3 数据关联关系（已实现）

- **客户-课题组-单位层级**：单位 → 课题组 → 客户
- **账单关联**：每个账单同时包含 customerId、groupId、companyId
- **出库单关联**：出库单通过 billId 关联到账单
